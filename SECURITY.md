# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please email us at hmisahaq01@gmail.com instead of using the issue tracker. Please include:

1. A description of the vulnerability
2. Steps to reproduce (if applicable)
3. Potential impact
4. Suggested fix (if you have one)

We take security seriously and will respond to your report as soon as possible.

## Supported Versions

| Version | Supported        |
| ------- | ---------------- |
| 1.x     | ✅ Supported     |
| < 1.0   | ❌ Not Supported |

## Security Best Practices

When using this package in your application:

1. **Keep Dependencies Updated**: Regularly update this package and its dependencies
2. **Validate Input**: Always validate user input before passing to barcode generation
3. **Secure Output**: Handle generated barcodes securely
4. **Use HTTPS**: When transmitting barcodes, always use HTTPS
5. **File Permissions**: Set appropriate file permissions when storing generated barcodes

## Known Security Issues

None currently reported.

## Disclosure Policy

- We'll investigate legitimate security reports
- We'll keep the report confidential until a fix is available
- We'll credit the researcher when publicly disclosing the issue (if they wish)

Thank you for helping keep this project secure!
